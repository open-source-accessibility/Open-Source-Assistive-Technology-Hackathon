## How to Contribute

1. Find your assigned issue (Issues tab ‚Üí filter by "Assignee: me")
2. Read the issue description ‚Äî it tells you which file to edit and what to fix
3. Click the file ‚Üí the edit pencil button ‚Üí make your change
4. Commit to a new branch with a descriptive name: `fix/[your-name]-[issue-number]`
5. Open a pull request using the PR template
6. **Wait for the bot** (~30 seconds) ‚Äî it will check your changes and provide feedback
7. **Read bot feedback carefully** ‚Äî it explains any issues and links to resources
8. Make changes if needed based on feedback
9. Request a review from your assigned peer reviewer
10. Respond to review feedback and make final improvements
11. Merge when approved!

See the [workshop documentation](../../docs/) for full step-by-step guidance.

## ü§ñ About the Automation Bot

When you open a pull request, an automated bot will:

- ‚úÖ **Check your changes** against quality standards
- ‚ôø **Validate accessibility** (heading hierarchy, link text, alt text, broken links)
- üìö **Provide educational feedback** explaining why things matter
- üîó **Link to resources** for learning more
- üè∑Ô∏è **Apply labels** to help facilitators prioritize reviews

**The bot is not a substitute for peer review!** It provides instant technical feedback so your human reviewers can focus on higher-level improvements, creative suggestions, and encouragement.

### How to Interact with the Bot

**Reading Bot Comments:**
- Bot posts ONE comment that updates as you make changes
- Use headings to navigate (Required Checks, Suggestions, Accessibility Analysis, Resources)
- Each issue includes file/line numbers and how to fix it

**Getting Help from the Bot:**
Comment on your PR with keywords:
- `@bot help` ‚Äî Get help information
- Mention "merge conflict" ‚Äî Get conflict resolution guide  
- Mention "request review" ‚Äî Get instructions for requesting human review

**When You Disagree:**
The bot isn't always right! If you believe feedback is incorrect:
1. Comment explaining your reasoning
2. Request human review
3. Facilitators can override bot checks
